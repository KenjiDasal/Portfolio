IMPLEMENTATION AND EVALUATION

Having the idea for the project in my head I decided to first familiarize myself and study with the concept of sound, in general, to better understand how amplitude and different frequencies of audio work. Then to get to know the sound library of p5. Watching multiple videos from “The Coding Train” on YouTube helped me on the beginnings. I started my coding while supporting myself with the information on specific references and libraries on p5js.org website. For a foundation, I embedded one song and created the first shape based on FFT function of the library, which analyze the frequency of the sound and returns an array. The first shape is using the analyze(), which allowed me to work with the array of amplitude values between 100 and 250. Using sine and cosine wave as per one of the tutorials watched on YouTube I created a round shape using an equation to make a generic wave visualizer.

I then moved on to the image I used the image() to show the image I made it so that it takes it from an array in the preload function. Then I tested it so that the images change and revert when Amp of the music. I also tested this effect on the on the colors of the visualizers and later on the particles. I wanted to make the image increase size without changing the height and width when the Amp is above 200 however this proved to be a problem and so I settled on making the image rotate back and forth from -0.5 to 0.5. I added a black fill on top of the image and gave it a opacity that changes according to the Amp When Making the particles is used and tweaked a file from the space invaders. One of the tweaks is using p5.Vector.random2d().mult(); this multiplies the vector by the amount in this case I used 50 this creates the circle in which the particle starts at. And I used copy() which copies the vector and again used the mult() on starting acceleration but it is multiplied by a random number between 0.0001 and 0.00001. I felt like something is still missing from the created visualization. I have decided to use one of the features I found in one of the examples founded in my inspirations. I made it so that the next two visuals based on the demo 2 and demo 3 from my inspirations. At this point I was happy with my progress and I want to start on interactions I first start with making key pressed function for the songs the is preloaded. The keys started off with 1 – 4 to demonstrate and see is the previous functions like images and colour will work due to the different amp ranges each song have.
Then I started making it so the song doesn’t overlap and able to pause and continue a song. Then I made buttons in order for interactions for touchscreen and mouse this acts much like the keys. Then I studied more in to p5 GUI in order to add a volume array and implement it in on the music. In order for this to work I used getVolume(); this allowed me to alter the volume of the music but for the project instead of hard coding a volume I made it so its connected to the params of the GUI. As Overall, I think this project went well, quick, fun and tough project to make. I had to reduce the amount of feature to add on the project for two reasons and these are:

  • - 1. Because of the days given for the project, report, video and website along with another CA, I was forced to reduce the features to make time this is even with the time I’ve received before the break.
  • - 2. Some features were troublesome especially when trying to add and alter effects of the music as an example I wanted to add a feature to increase the bass of the music however this needed me to learn EQ from the sound library but I don’t have the time.

If I had more time, I would like to add more features to the project, for example, letting the user choose between import music or connect external audio source like Spotify to it and let the user actually choose any song he wants it would be really beneficial for the overall project.
